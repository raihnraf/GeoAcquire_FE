## Interaction Patterns

### Filter by Status Flow

1. User clicks Filter button in header
2. Filter bar expands/slides down below header
3. User clicks status buttons (Free, Negotiating, Target)
4. Selected buttons show active color (filled)
5. URL updates: `?status=free,target`
6. Map filters to show only matching parcels
7. Status bar updates count: "Showing 12 parcels"
8. User clicks "Clear Filters" to reset
9. URL clears, all parcels shown

### Bounding Box Filter Flow

1. User clicks "Draw Box" button in filter bar
2. Map cursor changes to crosshair
3. Mode badge appears: "Drawing Box"
4. User clicks and drags on map
5. Blue rectangle appears while dragging
6. On release: rectangle persists, parcels filter
7. URL updates: `?bbox=106.8,-6.2,106.9,-6.1`
8. Clear button (X) appears at top-right of rectangle
9. User clicks X or mode badge X to exit box mode
10. Rectangle and filter clear

### Buffer Analysis from Selected Parcel Flow

1. User clicks a parcel on map
2. Sidebar opens with parcel details
3. User clicks "Analyze Nearby" button
4. Buffer panel appears with distance input (default 500m)
5. User adjusts distance (optional)
6. User clicks "Apply" button
7. Buffer circle appears on map centered on parcel
8. Nearby parcels highlight in blue
9. URL updates: `?buffer=123:500`
10. Sidebar updates with results list
11. User clicks result to view parcel details
12. User clicks X in buffer panel or mode badge to exit

### Buffer Analysis from Point Flow

1. User clicks "Analyze Area" button in header
2. Map cursor changes to crosshair
3. Mode badge appears: "Select Center"
4. Buffer panel appears with distance input
5. User clicks anywhere on map
6. Blue pin appears at clicked location
7. Buffer circle appears centered on pin
8. Nearby parcels highlight in blue
9. URL updates: `?buffer=-6.2,106.8:500`
10. Sidebar opens with results list
11. User can click different location to recenter
12. User clicks X to exit buffer mode

### Share Filtered View Flow

1. User applies filters or draws buffer
2. User clicks Share button in header
3. Current URL copied to clipboard
4. Toast appears: "Link copied to clipboard"
5. User shares link with colleague
6. Colleague opens link
7. App parses URL params
8. Map restores exact filter/buffer state

### Clear All Filters Flow

1. User has active filters (status, bbox, or buffer)
2. User clicks "Clear Filters" button
3. All filters reset to default
4. Map shows all parcels
5. URL params cleared
6. Toast: "Filters cleared"

## Implementation Notes

1. **Filter State Management:** Use URL params as single source of truth
   - Parse URL params on mount
   - Update URL params on filter change
   - Use `useSearchParams` from React Router or custom hook

2. **Buffer Visualization:** Use Leaflet Circle for buffer radius
   ```typescript
   L.circle([lat, lng], { radius: bufferDistance, ... })
   ```

3. **Bounding Box:** Use Leaflet Rectangle
   ```typescript
   L.rectangle([[south, west], [north, east]], { ... })
   ```

4. **Parcel Filtering:** Filter data before passing to ParcelLayer
   - Status filter: simple array filter
   - Bbox filter: use ` turf.booleanPointInPolygon` or custom bounds check
   - Buffer filter: use distance calculation or backend endpoint

5. **Mode State:** Track active mode in parent component (MapView or App)
   - Modes: `normal`, `bbox`, `buffer-point`, `buffer-parcel`
   - Only one mode active at a time
   - Exit mode on Escape key

6. **Icons (lucide-react):**
   - Filter for filter button
   - BoxSelect for bounding box mode
   - Target/Radio for buffer mode
   - MapPin for buffer center marker
   - Share2 for share button
   - SearchX for no results
   - X for clear/close actions

7. **Component Extensions Required:**
   - MapHeader: Add filter toggle, share button, buffer mode button
   - MapView: Add bbox drawing, buffer point selection, mode state
   - ParcelSidebar: Add "Analyze Nearby" button, buffer results display
   - ParcelLayer: Add highlight styles for buffer results

**New Components to Create:**
- FilterBar: Status filter buttons with active states
- BufferPanel: Floating panel with distance input and apply/cancel
- ModeBadge: Floating indicator for current map mode
- BufferResultsList: Sidebar content for nearby parcels

**URL Parameter Parsing:**
```typescript
// Parse bbox parameter
const parseBbox = (bbox: string): L.LatLngBounds => {
  const [minLng, minLat, maxLng, maxLat] = bbox.split(',').map(Number)
  return L.latLngBounds([minLat, minLng], [maxLat, maxLng])
}

// Parse buffer parameter
const parseBuffer = (buffer: string): { center: L.LatLng; radius: number } => {
  const [center, radius] = buffer.split(':')
  const [lat, lng] = center.split(',').map(Number)
  return { center: L.latLng(lat, lng), radius: Number(radius) }
}
```
